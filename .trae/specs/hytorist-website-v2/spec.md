# Hytorist 液压扳手网站 - 产品需求文档

## 概述
- **摘要**: 为北京海拓瑞斯机电科技有限公司创建一个现代化、功能完整的企业网站，包含产品展示、询盘功能和后台管理系统。
- **目的**: 替换老旧的ASP网站，提供现代化的用户体验，支持询盘功能，便于客户与公司业务对接。
- **目标用户**: 工业客户、采购商、潜在合作伙伴

## 目标
- 提供响应式、现代化的UI设计
- 实现完整的产品展示功能
- 实现询盘表单提交和管理
- 提供后台管理界面管理询盘
- 确保系统稳定性和可扩展性

## 非目标（范围外）
- 在线支付功能
- 用户账户系统
- 复杂的产品筛选和搜索（v3版本）
- 多语言支持（v3版本）

## 背景与上下文
- 现有网站使用ASP技术，版本老旧，用户体验差
- 公司主要产品为液压扳手及相关设备
- 需要支持客户询盘功能，便于业务对接
- 技术栈选择：React + TypeScript + Tailwind CSS（前端），Node.js + Express + SQLite（后端）

## 功能需求
- **FR-1**: 首页展示 - 英雄区、公司优势、特色产品展示
- **FR-2**: 产品展示 - 液压扳手及相关产品列表展示
- **FR-3**: 询盘功能 - 客户可提交询盘表单
- **FR-4**: 询盘管理 - 后台查看和管理询盘
- **FR-5**: 公司介绍 - 关于我们页面
- **FR-6**: 联系我们 - 联系方式展示

## 非功能需求
- **NFR-1**: 响应式设计 - 支持桌面和移动设备
- **NFR-2**: 性能 - 页面加载时间 < 3秒
- **NFR-3**: 可用性 - 界面简洁易用，表单验证完善
- **NFR-4**: 可维护性 - 代码结构清晰，易于扩展

## 约束条件
- **技术**: 使用React + TypeScript + Tailwind CSS作为前端，Node.js + Express + SQLite作为后端
- **业务**: 保留原网站的版权信息和备案号
- **依赖**: 无外部付费API依赖

## 假设条件
- 用户有基本的网络浏览器使用经验
- 后端服务器运行在稳定的环境中
- SQLite数据库足以支撑当前业务规模

## 验收标准

### AC-1: 首页完整展示
- **给定**: 用户访问网站首页
- **当**: 页面加载完成
- **然后**: 显示英雄区、公司优势、特色产品三个主要部分
- **验证**: `human-judgment`

### AC-2: 产品列表展示
- **给定**: 用户访问产品页面
- **当**: 页面加载完成
- **然后**: 显示所有产品，每个产品包含名称、分类、描述
- **验证**: `human-judgment`

### AC-3: 询盘表单提交
- **给定**: 用户填写完整的询盘表单
- **当**: 用户点击提交按钮
- **然后**: 表单数据保存到数据库，显示成功提示
- **验证**: `programmatic`

### AC-4: 询盘表单验证
- **给定**: 用户提交不完整的询盘表单
- **当**: 用户点击提交按钮
- **然后**: 显示相应的错误提示，阻止提交
- **验证**: `programmatic`

### AC-5: 询盘管理后台
- **给定**: 管理员访问询盘管理页面
- **当**: 页面加载完成
- **然后**: 显示所有询盘列表，支持查看详情和更新状态
- **验证**: `programmatic`

### AC-6: 响应式布局
- **给定**: 用户在不同设备上访问网站
- **当**: 调整浏览器窗口大小或使用移动设备
- **然后**: 页面布局自动调整，保持良好的用户体验
- **验证**: `human-judgment`
